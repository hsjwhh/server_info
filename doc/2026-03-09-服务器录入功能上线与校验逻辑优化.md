# 服务器入库功能上线与硬件校验逻辑重构 (v1.0.0)
日期: 2026-03-09

## 1. 概述
本次更新标志着项目正式进入 **v1.0.0** 阶段。核心目标是完善服务器入库流程，统一全项目的硬件数据处理标准，并优化开发环境的稳定性。

## 2. 新增功能：服务器录入页面
在 `src/pages/server/ServerEntryPage.vue` 中实现了全新的录入系统：
- **逻辑分组布局**：采用“两行两列”卡片结构（标识、核心硬件、存储清单、扩展系统），在宽屏下平铺显示，窄屏自动堆叠。
- **智能化硬件联动**：
    - **CPU 自动补全**：升级为带实时建议的搜索框，显示 TDP、核心数及 Socket 信息。
    - **主板自动过滤**：选中 CPU 后，主板下拉框仅显示与其 Socket 兼容的型号。
- **多硬件动态录入**：
    - 存储（M.2/SSD/HDD）与扩展卡（GPU/LAN/RAID）支持点击“添加项”动态增减。
    - 内存（RAM）固定为单种型号录入，但支持数量调整。
- **数据序列化**：提交时自动将动态列表合并为分号（`;`）分隔的字符串，完美适配数据库 `text` 字段存储需求。

## 3. 技术重构与底层优化
### 3.1 硬件工具类提取 (`src/utils/hardware.js`)
- 提取了通用的 `formatSocket()` 函数。
- 实现了标准化的 Socket 名称转换逻辑（如 `FCLGA4677` → `LGA-4677`），消除了项目中多处手写正则的冗余。

### 3.2 校验层增强 (`src/stores/configPlanValidation.js`)
- **三级反馈体系**：校验结果细分为 `blockers` (阻断)、`warnings` (警告)、`infos` (提示)。
- **正向反馈逻辑**：当配置全部通过且完整时，会显示“兼容性检查通过”的正向提示。
- **安全性防护**：
    - 修复了 Intel `FCLGA` 前缀在特定场景下的正则误判。
    - 引入了 `warnIfRef` 检查，在开发环境下主动提醒开发者避免误传 Vue Ref 对象，提升代码稳定性。

### 3.3 UI/UX 细节对齐
- **视觉统一**：同步了全项目的标题样式（`.page-title`）、页面间距（`gap: 1.5rem`）及圆角规范。
- **录入优化**：将 BMC 密码输入框改为普通文本，方便管理员快速核对初始密码。
- **对齐修正**：通过占位符技术，确保内存项与下方带有删除按钮的存储项在视觉上精确对齐。

## 4. 环境与工程化
- **跨域解决方案**：在 `vite.config.js` 中配置了 `server.proxy`，将 `/api` 请求转发至 `localhost:3000`，彻底解决开发环境下的 CORS 限制。
- **版本归档**：重整了 `CHANGELOG.md`，将前期所有基建工作（Auth 系统、Vuestic UI 迁移、安全修复等）统一归档为 1.0.0 基准版本。
